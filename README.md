# ParcerTG — охотник за горячими лидами в Telegram

Сервис от имени отдельного Telegram-аккаунта слушает выбранные русскоязычные чаты и каналы, локально оценивает сообщения по прозрачным правилам, удаляет дубли и мгновенно присылает подходящие заказы в личного Telegram-бота.

## Возможности

- готовый каталог из 100 русскоязычных Telegram-источников;
- мониторинг десятков и сотен чатов и каналов через Telethon/MTProto;
- rule-based scoring от 0 до 100 без передачи сообщений в LLM;
- фильтрация резюме, рекламы, обучения, бартера и работы «за процент»;
- приоритет Python, FastAPI, Telegram-ботов, AI-интеграций, CRM, API и автоматизации;
- точная и fuzzy-дедупликация репостов за настраиваемое окно;
- карточка горячего лида с причиной оценки и ссылкой на оригинал;
- кнопки «Взял в работу», «Не подходит» и «Спам»;
- команда `/stats`;
- отчёт при запуске: настроено, подключено и недоступно источников;
- PostgreSQL в Docker, SQLite для локальной разработки;
- тесты скоринга, дедупликации и каталога источников.

## Важное ограничение Telegram

Проект намеренно не отправляет тексты сообщений в LLM и не использует их для обучения моделей. Telegram API Terms запрещают использовать или агрегировать данные Telegram для обучения, улучшения и развёртывания AI/ML-систем. Перед эксплуатацией проверьте актуальные условия Telegram API и законодательство вашей юрисдикции.

## Быстрый запуск

### 1. Получите MTProto-ключи

Создайте приложение на `my.telegram.org` и получите `api_id` и `api_hash`. Используйте отдельный Telegram-аккаунт.

Аккаунт должен быть подписан на каналы и состоять в чатах, обновления из которых нужно получать. Проект намеренно не вступает автоматически сразу в 100 сообществ: массовое автоматическое вступление создаёт риск FloodWait и ограничений аккаунта.

### 2. Создайте бота уведомлений

Создайте бота через `@BotFather`, откройте с ним диалог и получите свой numeric Telegram ID, например через `@userinfobot`.

### 3. Подготовьте окружение

```bash
cp .env.example .env
```

Заполните `TELEGRAM_API_ID`, `TELEGRAM_API_HASH`, `BOT_TOKEN`, `ADMIN_IDS` и `NOTIFY_CHAT_ID`.

Каталог `config/sources.txt` уже содержит 100 источников, поэтому вручную переносить их в `.env` не требуется.

### 4. Создайте StringSession

Локально:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .
python -m scripts.generate_session
```

Или через Docker после заполнения `TELEGRAM_API_ID` и `TELEGRAM_API_HASH`:

```bash
docker compose run --rm app python -m scripts.generate_session
```

Скопируйте полученную строку в `TELEGRAM_SESSION`. Она даёт доступ к аккаунту — не публикуйте её и не коммитьте `.env`.

### 5. Запустите

```bash
docker compose up -d --build
docker compose logs -f app
```

При успешном запуске в логе появится:

```text
collector_started configured=100 resolved=... unavailable=...
```

Недоступные username выводятся отдельными сообщениями `chat_source_unavailable` и не останавливают мониторинг остальных источников.

## Настройка источников

Стандартный каталог:

```env
CHAT_SOURCES_FILE=config/sources.txt
```

В `config/sources.txt` можно использовать по одному значению на строку:

```text
# Комментарий
freelance_rabota_chat
https://t.me/job_for_bots
-1001234567890
```

Пустые строки и комментарии игнорируются. Повторы удаляются без учёта регистра.

Дополнительные источники можно добавить через `.env`, не меняя основной каталог:

```env
CHAT_SOURCES=my_private_chat,-1001234567890
```

Источники из `CHAT_SOURCES` добавляются к значениям из `CHAT_SOURCES_FILE`. Для приватных групп аккаунт должен быть участником.

## Скоринг

По умолчанию уведомление приходит при `MIN_LEAD_SCORE=65`.

Пример начисления:

- +30 — явно ищет разработчика или исполнителя;
- +20 — Python, FastAPI, Telegram, AI, CRM, API, PostgreSQL, Redis, Docker;
- +18 — требуется разработка, интеграция или автоматизация;
- +15 — бюджет или готовность платить;
- +10 — срочность;
- отрицательные баллы — резюме, реклама услуг, курсы, бартер, оплата за долю, только офис.

Правила находятся в `app/scoring.py` и легко расширяются под собственные услуги.

## Локальная разработка

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e '.[dev]'
pytest
ruff check .
```

Для SQLite укажите:

```env
DATABASE_URL=sqlite+aiosqlite:///./parcertg.db
```

## Безопасность и антиспам

- сервис только уведомляет владельца и не пишет заказчикам автоматически;
- не используйте один StringSession на нескольких одновременно работающих экземплярах;
- ограничьте доступ к серверу и `.env`;
- не добавляйте чаты и данные, к которым у аккаунта нет законного доступа;
- соблюдайте правила групп, Telegram API Terms и требования к персональным данным.
