# Бесплатный старт через Telemetrio

## Что даёт бесплатный тариф

На странице тарифов Telemetrio для Free указаны одно отслеживание,
неограниченное количество слов и каналов внутри отслеживания, лимит
100 упоминаний и уведомления в Telegram.

Этого достаточно, чтобы без затрат проверить спрос и откалибровать фильтры
ParcerTG.

## Настройка

1. Зарегистрируйтесь в Telemetrio.
2. Откройте инструмент отслеживания событий/упоминаний.
3. Создайте одно отслеживание.
4. Добавьте строки из `config/telemetr_keywords.txt` в ключевые слова и фразы.
5. Добавьте строки из `config/telemetr_minus_words.txt` в минус-слова.
6. Подключите Telegram-уведомления через актуального бота
   `@TelemetrioAlertBot`.
7. Убедитесь, что тестовое уведомление появляется в этом диалоге.

Telemetrio выполняет широкий поиск, а ParcerTG отвечает за второй слой:

- оценку намерения нанять;
- проверку совпадения с целевым стеком;
- отсечение резюме и рекламы;
- дедупликацию;
- учёт результата.

## Автономный режим

В `.env` укажите:

```env
SOURCE_PROVIDERS=telegram_web
TELEGRAM_WEB_TARGET_CHAT=TelemetrioAlertBot
```

После одноразовой авторизации Telegram Web ParcerTG будет самостоятельно читать
новые уведомления из `@TelemetrioAlertBot`. Ручная пересылка не требуется.

Полная настройка QR-входа:
[`TELEGRAM_WEB_SETUP.md`](TELEGRAM_WEB_SETUP.md).

## Ручной резерв

Даже при включённом автосборе уведомление можно переслать в ParcerTG вручную.
Отдельная команда не требуется.

Для текста, полученного вне Telemetrio:

```text
/lead Нужен Telegram-бот для обработки заявок и интеграции с CRM. Бюджет 80 000 ₽.
```

Для проверки качества фильтра без сохранения:

```text
/check Нужен Python-разработчик для FastAPI, начать на этой неделе.
```

## Когда переходить на платный источник

Переход к TGStat Callback имеет смысл после одного из событий:

- бесплатные 100 упоминаний закончились слишком быстро;
- интерфейс Telegram Web стал нестабилен;
- найден хотя бы один оплаченный заказ;
- подтверждена приемлемая доля релевантных сообщений.
